# PRD — Local-LLM Hands-on Monorepo

**Status:** v1 (specs ready to implement) · **Owner:** course author · **Build agents:** Claude Code (best-practice path) and/or opencode + LM Studio (open path)

## 1. Problem & goal
Developers learn local LLMs best by building. This monorepo is a set of small, runnable, **provider-swappable** apps over a public, anonymized retail-reviews dataset. Goal: every example runs on a **local model via the OpenAI-compatible endpoint** and switches to any cloud provider by changing `base_url` + `api_key`.

## 2. Non-goals
- No production deployment, auth, or scaling work (it's educational).
- No model training/fine-tuning in v1 (Unsloth is referenced, not used).
- **No company names, no PII, no proprietary data — ever.**

## 3. Users
- Software engineers (want the code + the agentic build workflow).
- Technical analysts (want the dashboards + the prompt/RAG patterns).

## 4. Principles
1. **One swap line.** The only code that knows local-vs-cloud is the client constructor.
2. **Structured output everywhere.** Every model call that feeds software returns validated JSON (Pydantic).
3. **Smallest tool that works.** Show when a DistilBERT classifier beats a 24B LLM.
4. **Reproducible.** Public datasets, pinned package versions, runs on a 48 GB Mac.
5. **PRD-first.** Each package has a spec an agent can implement with minimal clarification.

## 5. Shared technical decisions
- **Engine:** LM Studio at `http://localhost:1234/v1` (Anthropic-compat `:1234` for Claude Code).
- **Backend:** Python 3.11 + FastAPI + `openai` SDK + `pydantic` v2. (.NET variant noted per package.)
- **Frontend:** Next.js (React) + a small component set; charts via a lightweight lib.
- **RAG:** `chromadb` + local embeddings via `/v1/embeddings`.
- **Lightweight NLP:** Hugging Face `transformers` (DistilBERT) for the classifier comparison.
- **Config:** a single `.env` with `LLM_BASE_URL`, `LLM_API_KEY`, `LLM_MODEL` consumed everywhere.

## 6. Config contract (every package imports this)
```python
# common/llm.py
import os
from openai import OpenAI
def get_client() -> OpenAI:
    return OpenAI(
        base_url=os.environ.get("LLM_BASE_URL", "http://localhost:1234/v1"),
        api_key=os.environ.get("LLM_API_KEY", "lmstudio"),
    )
LLM_MODEL = os.environ.get("LLM_MODEL", "local-model")
```
Switching to OpenAI = set `LLM_BASE_URL=https://api.openai.com/v1`, a real `LLM_API_KEY`, and a real `LLM_MODEL`. Nothing else changes.

## 7. Packages & acceptance (summary)
| Pkg | Name | Done when |

| 00 | serve-lmstudio | `python hello.py` and `curl` both return a completion from the local server |

| 06 | agentic-capstone (P6) | package 04 backend regenerated by an agent from `AGENTS.md` + a `SKILL.md` |

## 8. Cross-cutting acceptance
- Every package has a README with run steps and a "swap to cloud" note.
- Every model call uses structured output where it feeds code.
- No secrets committed; `.env.example` provided.

## 9. Risks
- **Model drift:** names/benchmarks change monthly → keep `model-table.md` authoritative.
- **Local quality variance:** small models may need more few-shot / tighter schemas; document fallbacks.
- **Dataset licensing:** use only the listed public datasets; keep attribution.
