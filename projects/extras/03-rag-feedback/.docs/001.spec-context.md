# 001 · Spec-Context — 03-rag-feedback (P3)

One-shot implementable context for the Product-Feedback RAG Assistant. Derived from
[`../../p3-feedback-rag.md`](../../p3-feedback-rag.md) and the repo CLAUDE.md
Projects contract.

## Prompt segments (the LLM call)

Layered, static-first so the static prefix is cache-friendly (article B3):

1. **SYSTEM** — "Answer questions about product feedback using ONLY the provided
   context. Each context item has an id; cite the ids you used. If the context does
   not contain the answer, say you don't know and return an empty citations list.
   Return ONLY JSON matching `{answer, citations:[{id, snippet}], confidence}`."
   (`backend/app.py::SYSTEM_PROMPT`)
2. **CONTEXT** — the top-k retrieved feedback docs, each as `[id] (category) text`
   (`backend/app.py::_format_context`). Built per query from Chroma retrieval.
3. **USER** — the dynamic question, last.

Request uses `response_format={"type": "json_object"}` (JSON mode). The response is
validated against `Answer`. On failure: retry once with the validator error
appended; a second failure raises `QuarantineError`, which the endpoint turns into
an honest "I don't know". All of this is `common.llm.structured_call` — not
reimplemented here.

**Grounding guard (B8):** after a successful parse, `answer_question` drops any
citation whose `id` was not in the retrieved set, so the answer can only cite the
supplied context. Empty retrieval short-circuits to "I don't know" before any LLM
call.

## Tech stack

- Backend: Python 3.11 · FastAPI · `openai` SDK (via `common/llm.py`) · pydantic v2.
- Vector store: `chromadb` — `EphemeralClient` (in-memory), bundled default
  embedder (local ONNX MiniLM, downloaded once then offline). Package-local
  `requirements.txt`; not in the shared venv. The embedding function is injectable
  (`build_collection(embedding_function=...)`).
- Offline embedder: `HashingEmbeddingFunction` (`backend/index.py`) — a
  deterministic bag-of-words hashing vectorizer with stopword removal, used by tests
  and offline demos so retrieval is reproducible with zero network.
- Frontend: Next.js 14 (App Router) + TypeScript — scaffold only.
- Tests: pytest + FastAPI `TestClient` (httpx). Fully offline; LLM client mocked,
  stub embedder injected via `dependency_overrides`.

Note: `models.py`/`index.py` use `typing.List/Optional[...]` rather than `X | None`
so the package imports on the host's Python 3.9 toolchain while remaining correct on
the 3.11 target.

## Data contract

Fixture `backend/fixtures/feedback.jsonl` — 20 synthetic feedback rows:
`{id: str, text: str, product_category: str}`. Several rows are planted to make
retrieval tests deterministic (e.g. `fb-001/008/015/016` describe shrinkage,
`fb-012` is the waterproof rain jacket). In the spirit of the Women's E-Commerce
Clothing Reviews schema [R-039] and the public Turkish dataset family [R-040],
[R-041]; no source-dataset rows, no PII (synthetic, generic fashion-retail).

Output `Answer`: `{answer: str, citations: list[Citation], confidence:
low|med|high}`, where `Citation = {id: str, snippet: str}`. `AskRequest`:
`{question: str, top_k: int (1..10, default 4)}`.
