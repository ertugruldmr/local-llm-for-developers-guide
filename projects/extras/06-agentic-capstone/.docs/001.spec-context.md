# 001 · Spec-Context — 06-agentic-capstone (P6)

One-shot implementable context for the Agentic Capstone. Derived from
[`../../p6-agentic-capstone.md`](../../p6-agentic-capstone.md) and the repo CLAUDE.md
Projects contract.

> **Design-goal framing (guardrail):** the agent's "minimal steering" /
> self-improving behavior is a **design goal of the workflow, not a benchmarked
> guarantee.** Document the run; make no general claim.

## What this package is

The deliverable is a **documented agent run**, not new runtime code. An agent —
Claude Code (best-practice path) or opencode (open path) — driven by this package's
[`AGENTS.md`](../AGENTS.md) + [`SKILL.md`](../SKILL.md) regenerates the **P1 backend**
(`../04-fullstack-analyzer/backend/`) from [`../../p1-review-analyzer.md`](../../p1-review-analyzer.md)
into `generated/backend/`. The teaching content is *where the human had to steer*.

## Inputs the agent reads

1. **AGENTS.md** — the config contract, structured-output / retry-once-then-quarantine
   pattern, build order, offline-tests rule, no-PII rule, type-hint rule.
2. **SKILL.md** — `scaffold-fastapi-from-spec`: the step-by-step procedure.
3. **The target spec** — `../../p1-review-analyzer.md`.

## Tech stack

- Agent host: Claude Code (Anthropic-compat local endpoint) and/or opencode (LM
  Studio). Same `AGENTS.md` + `SKILL.md` for both.
- Generated target: Python 3.11 · FastAPI · `openai` SDK (via `common/llm.py`) ·
  pydantic v2 · pytest (offline, client mocked) — i.e. P1's stack.
- This package's own code: `backend/run_record.py` — the `AgentRunRecord` schema +
  an `evaluate(generated_dir)` helper.

Note: `run_record.py` uses `typing.Optional` / explicit `list[...]` so it imports on
the host's Python 3.9 toolchain.

## Contract (the run record)

`backend/run_record.py`: `SteeringMoment {step, issue, human_action, autonomous=False}`
and `AgentRunRecord {path, model, steps_total, steps_autonomous, steering_moments[],
backend_tests_pass}`. `evaluate(generated_dir)` runs the generated backend's pytest
and fills `backend_tests_pass`.
