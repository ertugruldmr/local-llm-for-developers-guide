# 001 · Spec-Context — 05-conversational-survey (P2)

One-shot implementable context for the Conversational AI-Survey Bot. Derived from
[`../../p2-conversational-survey.md`](../../p2-conversational-survey.md) and the repo
CLAUDE.md Projects contract.

## Prompt segments (the LLM call)

Layered, static-first so the static prefix is cache-friendly (article B3):

1. **SYSTEM** — adaptive-survey instructions: probe the fixed taxonomy
   `[sizing, fabric, fit, delivery, price, service]`, ask ≥`min_follow_ups` and
   ≤`max_follow_ups`, decide ask-vs-finish each turn, return ONLY JSON
   (`FollowUpDecision`); when finishing, return a `SurveySummary`
   (`backend/app.py::SYSTEM_PROMPT`).
2. **RUNNING_SUMMARY** — compacted older turns (B6), prepended once the verbatim
   window exceeds `window_turns`. Static-ish → cache-friendly.
3. **TURNS** — the last `window_turns` verbatim turns (B5 short-term memory), last.

Each agent step (`FollowUpDecision`) and the final `SurveySummary` are JSON-mode
calls validated through `common.llm.structured_call` (retry-once-then-quarantine).
Not reimplemented here.

## Tech stack

- Backend: Python 3.11 · FastAPI · `openai` SDK (via `common/llm.py`) · pydantic v2.
- Session store: in-memory dict keyed by `session_id` (stretch: SQLite).
- Frontend: Next.js 14 (App Router) — chat panel + running-summary side panel; not
  yet scaffolded.
- Tests: pytest; offline (LLM client mocked, same FakeClient pattern as P1).

Note: `models.py` uses `typing.Optional[...]` rather than `X | None` so the package
imports on the host's Python 3.9 toolchain while remaining correct on the 3.11 target.

## Data contract

Fixtures (synthetic, generic fashion-retail, no PII):
- `backend/fixtures/survey_config.json` — base question, taxonomy, `min/max_follow_ups`,
  `window_turns`, stop condition.
- `backend/fixtures/transcript.jsonl` — one canned return-scenario conversation used
  as the offline loop/summary fixture.

Output schemas (`backend/models.py`): `FollowUpDecision {action, question?, rationale}`,
`SurveySummary {topics[], sentiment, verbatims[<=5], next_action, summary(<=280)}`,
plus `SessionState`, `StartSessionResult`, `TurnRequest`.
