# 001 · Spec-Context — 04-fullstack-analyzer (P1)

One-shot implementable context for the Review / Survey Analyzer. Derived from
[`../../p1-review-analyzer.md`](../../p1-review-analyzer.md) and the repo CLAUDE.md
Projects contract.

## Prompt segments (the LLM call)

Layered, static-first so the static prefix is cache-friendly (article B3):

1. **SYSTEM** — "You label retail product reviews. Return ONLY JSON matching this
   schema {sentiment, themes[], sizing_issue, churn_risk, summary}. Pick themes only
   from the fixed list `[sizing, fabric, fit, color, delivery, price, quality,
   service]`. summary <= 20 words." (`backend/app.py::SYSTEM_PROMPT`)
2. **FEW-SHOT** — 2 static labeled (review → JSON) pairs (`backend/app.py::FEW_SHOT`,
   concept A3). Static so they precede the dynamic input.
3. **USER** — the dynamic review text (last).

Request uses `response_format={"type": "json_object"}` (JSON mode). The response is
validated against `ReviewInsight`. On failure: retry once with the validator error
appended; a second failure raises `QuarantineError` and the row is quarantined.
All of this is `common.llm.structured_call` — not reimplemented here.

## Tech stack

- Backend: Python 3.11 · FastAPI · `openai` SDK (via `common/llm.py`) · pydantic v2.
- Baseline: HF `transformers` DistilBERT
  (`distilbert-base-multilingual-cased-sentiments-student`) + `torch` — OPTIONAL,
  heavy, lazy-imported and guarded (`backend/compare.py`). Package-local
  `requirements.txt`; not in the shared venv.
- Frontend: Next.js 14 (App Router) + TypeScript — scaffold only.
- Tests: pytest + FastAPI `TestClient` (httpx). Fully offline; LLM client mocked.

Note: `models.py` uses `typing.Optional[...]` rather than `X | None` so the package
imports on the host's Python 3.9 toolchain while remaining correct on the 3.11 target.

## Data contract

Fixture `backend/fixtures/reviews.jsonl` — 20 synthetic rows mirroring the
Women's E-Commerce Clothing Reviews schema [R-039]:
`{review_text: str, rating: 1..5, recommended: bool, age: int, class_name: str}`.
No source-dataset rows, no PII (synthetic, generic fashion-retail).

Output `ReviewInsight`: `{sentiment: pos|neu|neg, themes: list[str],
sizing_issue: bool, churn_risk: low|med|high, summary: str(<=160 chars)}`.
