# Proposal — "Local LLMs for Developers" hands-on track

## One-liner
A short, reproducible set of full-stack apps that teach developers to run, code with, and build on local LLMs — every example swappable to any cloud provider by changing two lines.

## Why now
The open-vs-closed model gap has compressed to ~30 Elo on coding leaderboards, and serving a capable model locally now fits on a laptop. Teams can get the privacy and cost benefits of local inference without sacrificing much capability — but only if developers know the tools and patterns. This track closes that gap in an afternoon.

## What learners walk away with
1. A working local server (LM Studio) and the muscle memory of the **base-URL swap**.
2. An editor agent and a CLI agent wired to a local model.
3. Five small apps demonstrating **structured output, RAG, multi-turn agents, and the "smallest tool that works" principle**.
4. A repeatable **agentic build workflow** (AGENTS.md + skills) usable on real projects.

## Format
- 1–2h live session (concepts → demos → one project build).
- A long-form article (4 parts) for self-paced study.
- An interactive overview map for navigation and review.
- This monorepo for hands-on practice.

## Scope guardrails
Anonymized retail-survey domain; public datasets only; no PII or proprietary content — safe to publish openly.

## Success metrics
- A developer can serve a model and run the swap in <15 min.
- A developer can explain prompt vs context vs harness engineering and point to a tool for each.
- The repo runs end-to-end on a 48 GB Apple Silicon machine and on Colab.
